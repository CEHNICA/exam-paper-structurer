# questions.json 结构

顶层结构：

```json
{
  "source": {
    "school": "学校/教研组名称（印刷体标题里能看到就填，看不到就省略）",
    "title": "试卷标题，如 2025—2026学年度第1学期9月八年级数学月月清",
    "author": "命题教师（有就填）",
    "grade": "年级",
    "pages": 8,
    "source_photos": ["原始照片文件名，按页面顺序列出"],
    "note": "任何值得让后来者知道的说明；默认不做手写答案识别，不用在这里交代answer核对情况（除非用户特别要求提取了答案）"
  },
  "questions": [ /* 见下 */ ]
}
```

每道题一个对象，字段：

| 字段 | 说明 |
|---|---|
| `id` | 题号（整数，按试卷顺序） |
| `type` | `"choice"`（单选）/ `"multi_choice"`（多选，印刷体题干里会写"有多项符合题目要求"之类的字样）/ `"fill_blank"`（填空题）/ `"solving"`（解答题） |
| `section` | 大题标题，如 `"一、选择题"`，同一大题内保持完全一致的字符串（渲染时用它分节） |
| `points` | 分值（整数），试卷上没写就省略 |
| `stem` | 题干。多问的解答题里 `(1)(2)` 之间用真正的换行符 `\n` 分隔，不要用字面的反斜杠+n（这是 latexify 那一步最容易踩的坑，见下方"已知的坑"） |
| `options` | 选择题才有，`{"A": "...", "B": "...", "C": "...", "D": "..."}` |
| `image` | 这道题配图的相对路径，如 `"images/q02.jpg"`；没有配图就是 `null` |
| `images` / `image_labels` | 一道题有多张图（比如证明题的图1/图2，或者选择题每个选项各是一张图）时用这一对数组代替单张的 `image`，`image_labels` 与 `images` 按下标一一对应（如 `["图1","图2"]` 或 `["A","B","C","D"]`） |
| `answer` | **默认省略这个字段**——这个技能的目的是记录题目本身，不是识别手写答案。只有用户明确要求"把答案也记下来"时才填：选择题填字母（多选给多个字母如`"BD"`），填空题填最终数值/表达式，解答题一般留 `null`（不数字化学生的解题过程） |

## 完整示例（默认行为：不提取答案，节选自凤城高一数学质检卷）

```json
{
  "source": {
    "title": "2025级高一上学期质量检测一 数学试题",
    "grade": "高一",
    "source_file": "微信图片_20251028070013_223_61.jpg.pdf",
    "note": "题干/选项为印刷体。解答题未数字化学生的解题过程，只保留印刷题干与配图。"
  },
  "questions": [
    {
      "id": 9, "type": "multi_choice", "section": "二、选择题（多选）", "points": 6,
      "stem": "下列图象中，是函数图象的是（　）",
      "options": {"A": "见图A", "B": "见图B（不是函数图象）", "C": "见图C", "D": "见图D"},
      "images": ["images/q09_A.jpg", "images/q09_B.jpg", "images/q09_C.jpg", "images/q09_D.jpg"],
      "image_labels": ["A", "B", "C", "D"]
    },
    {
      "id": 17, "type": "solving", "section": "四、解答题", "points": 15,
      "stem": "某网店销售一批新款削笔器，进价为10元/个.经统计，该削笔器的日销售量$y$（单位：个）与售价$x$（单位：元）满足如图所示的函数关系.\n(1) 为了使这批削笔器的日利润最大，应怎样定制这批削笔器的销售价格？\n(2) 为了使这批削笔器的日利润不低于售价为15元时的日利润，求售价$x$的取值范围.",
      "image": "images/q17.jpg"
    }
  ]
}
```

## 用户明确要求提取答案时（可选，非默认）

`answer` 字段照常加回去，`source.note` 里说明核对方式：

```json
{
  "id": 2, "type": "choice", "section": "一、选择题", "points": 4,
  "stem": "如图，在四边形ABCD中，$AD\\parallel BC$，要使四边形ABCD成为平行四边形，则应添加的条件是（　）",
  "options": {"A": "$AB=CD$", "B": "$AC=BD$", "C": "$AD=AB$", "D": "$\\angle ABC+\\angle BCD=180^\\circ$"},
  "image": "images/q02.jpg", "answer": "C"
}
```
